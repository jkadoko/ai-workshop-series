# Setting Up the Antigravity IDE

## Overview

In previous modules, we installed Visual Studio Code (VS Code) as our primary code editor. The **Antigravity IDE** is the AI-powered developer surface that integrates directly into this editing environment (typically via a dedicated VS Code extension). 

While the command-line interface (CLI) is perfect for programmatic, automated, and batch workflows, the **IDE surface** is optimized for **editor-first workflows**—tasks where you are writing, reading, refactoring, and debugging code in real time.

```
┌──────────────────────────────────────────────────────────────────┐
│ VS Code (Antigravity IDE Surface)                                │
│                                                                  │
│ ┌──────────────────────┐ ┌───────────────────┐ ┌───────────────┐ │
│ │ Explorer             │ │ hello_script.py   │ │ Antigravity   │ │
│ │                      │ │                   │ │ AI Chat       │ │
│ │ 📄 hello_script.py   │ │ def greet():      │ │               │ │
│ │                      │ │   print("Hello")  │ │ Ask a question│ │
│ │                      │ │                   │ │ or give code  │ │
│ │                      │ │                   │ │ instructions  │ │
│ └──────────────────────┘ └───────────────────┘ └───────────────┘ │
└──────────────────────────────────────────────────────────────────┘
```

### Why Use the IDE Surface?

- **Context Awareness** — The IDE automatically knows which file you are looking at, where your cursor is positioned, and what other files are open in your workspace.
- **Inline Code Edits** — You can highlight a block of code and ask the agent to rewrite or fix it directly in the editor, without copying and pasting.
- **Interactive Chat Sidebar** — Discuss design patterns, ask for explanations of complex code, or generate boilerplate code without leaving the editor window.
- **Automatic Workspace Indexing** — The IDE indexes your project files, allowing you to ask high-level questions like, *"Where in our codebase do we validate requirements?"*

> **In a Regulated Context:** The IDE provides a highly collaborative environment for design and development, but the same rule applies: all code edits or comments generated by the AI must be reviewed and verified before committing them to version control.

## Setup & Configuration Steps

### 1. Enable the Antigravity IDE Extension

**Justification:** The IDE surface is delivered as an extension for VS Code that links your local editor to the Antigravity developer agent.

**Steps:**
1. Open **Visual Studio Code**.
2. Click the **Extensions** icon on the Activity Bar on the left side (or press `Ctrl+Shift+X`).
3. Search for **Antigravity**.
4. Click **Install**.
5. Once installed, you will see a new **Antigravity** icon (an agent symbol) on the Activity Bar.

---

### 2. Sign In and Authenticate

**Justification:** Just like the CLI, the IDE extension needs permission to communicate with the Google backend models.

**Steps:**
1. Click the **Antigravity** icon in the left Activity Bar to open the Antigravity sidebar.
2. Click the **Sign In** button.
3. Your web browser will open automatically, prompting you to log in with your Google account.
4. Follow the prompts to authorize the extension. Once completed, the Antigravity sidebar in VS Code will update to show you are connected.

---

### 3. Open Your Project Workspace

**Justification:** The Antigravity IDE requires an open workspace directory to index files and provide relevant context.

**Steps:**
1. Go to **File** > **Open Folder...**
2. Select the workshop repository folder (or your project directory).
3. If prompted, select "Yes, I trust the authors."
4. Wait a few moments. The Antigravity extension will start indexing your files in the background (you will see a small indexing indicator in the bottom status bar).

## Core IDE Workflows

### 1. Using the Chat Sidebar

The chat sidebar is your companion for general Q&A, documentation drafting, and conceptual discussions.

**Try It:**
1. Open the **Antigravity** chat panel from the left sidebar.
2. In the text input box, type:
   > How do I write a Python function to validate that an email address matches standard formats?
3. Press **Enter**. The agent will generate the Python code and explain the regular expression used right in the panel.

---

### 2. Inline Code Generation & Refactoring

You can prompt the agent to write or edit code directly inside your open files.

**Try It:**
1. In your explorer, open a Python file (e.g., `hello_vscode.py`).
2. Highlight a function or click on a blank line.
3. Press `Ctrl+I` (or right-click and select **Antigravity: Inline Edit**).
4. A small prompt input box will appear. Type:
   > Add a docstring and type hints to this function, following PEP 8 guidelines.
5. Press **Enter**. The agent will display a diff showing the proposed changes.
6. Click **Accept** (or press `Ctrl+Enter`) to apply the changes directly to your file, or **Reject** to discard them.

---

### 3. Explaining Code

When inheriting legacy code or working with unfamiliar libraries, you can ask the agent for a detailed walkthrough.

**Try It:**
1. Highlight any block of code in your editor.
2. Right-click and select **Antigravity: Explain Code**.
3. The explanation will render in the chat sidebar, explaining the input parameters, logic flow, and potential edge cases.

## IDE vs. CLI: Choosing the Right Surface

While both surfaces connect to the same core models, they serve different purposes:

| Feature / Workflow | Antigravity IDE | Antigravity CLI |
223: | **Primary Interface** | Graphical sidebar, inline diffs | Terminal command line |
224: | **Best For** | Active coding, refactoring, code explanation | Programmatic automation, scripting, batch tasks |
225: | **Integration** | Runs inside Visual Studio Code | Runs in PowerShell, cmd, or shell scripts |
226: | **Context Scope** | Active document, highlighted code, workspace files | Scoped by CLI parameters, piped input, or skills |
227: | **Automation** | Interactively guided by developer | Can run programmatically, on schedules, or in pipelines |
228: | **Auditing** | Manual review of changes before applying | Command logs, output files, script trails |
